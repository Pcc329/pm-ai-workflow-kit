# 環境設定的通用類別

> 這份只列「一個新專案通常會需要盤點哪幾類設定」，不含任何真實變數名稱或金鑰。
> 真實清單寫進該專案的 instance 資料夾。

## 一般會用到的設定類別

| 類別 | 說明 | 範例（不是真實值） |
|---|---|---|
| 資料庫 / 資料來源存取 | 連線位址、存取金鑰 | DB_URL、DB_TOKEN |
| AI / LLM API | 呼叫外部模型的金鑰 | 各家 AI 服務商的 API Key |
| 部署平台 | 前端/後端託管服務的專案設定 | 部署平台的專案 ID |
| 版本控制 | Repo 存取權限 | Personal Access Token |
| 第三方整合 | 任何串接的外部服務 | 依實際串接項目而定 |

## 安全原則（放諸四海皆準）

- 任何真實金鑰不寫死在程式碼裡，用環境變數或執行階段輸入（例如 `getpass()`）
- 任何會進版控的檔案（含 notebook），執行後檢查是否不小心把真實金鑰留在輸出結果裡
- 個人事務（例如語言學習練習、私人筆記）不混進工作用的 repo，即使技術上共用同一個帳號

## 交接時，環境設定該怎麼交接

- 不要把真實金鑰寫進交接文件——交接文件只列「需要跟誰要什麼」，實際金鑰當面/私訊交接
- 每個金鑰標明用途跟取得管道，讓接手的人知道金鑰過期或權限不足時該找誰
